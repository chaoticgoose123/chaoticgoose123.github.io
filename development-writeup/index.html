Interface Development

Introduction

To gain more experience with React, I developed a simple app that allows users to sort and filter a list of items, then add items to a collection. You can play with the result here.

Design Considerations

Besides the basic functionality described above, there were a few other aspects of the web app's design and implementation that I took into consideration:

For starters, I ensured that the app was accessible to assistive-device users. This meant using semantic HTML and default <input> elements where possible. I then evaluated the site with WebAIM's WAVE accessibility tool, and tested its functionality and ease of navigation using the VoiceOver screen reader.

Another important consideration was that of performance. React apps can be painfully slow when implemented poorly. Thus, I challenged myself to handle a list of >150 items, forcing me to look into ways to optimize my app. To spare you the details, two changes I made led to the biggest performance improvements: (1) eliminating redundant useEffect() calls and re-renders, and (2) toggling display for list items, instead of using conditional rendering.

Conclusion

Although highly contrived, working on this mini-project deepened my familiarity with React and increased my appreciation of the amount of work it takes to translate designs into fully-functional interfaces.
