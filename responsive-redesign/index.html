Responsive Redesign

Introduction

When redesigning a website, it's often useful to begin with a critical analysis of its current design. We can then use our findings to make targeted and tangible improvements, by iterating on prototypes that directly address its flaws.

To practice this workflow, I had a go at redesigning this delightfully 2000s website for Tri-State Camera, a camera and electronics store based in New York.

The Website

Here's the site in all its HTML-table-element, text-made-of-images glory:

It isn't the worst site out there, but it clearly hasn't been updated in a while. Plus, many of its interactions are unintuitive by today's standards, and its layout is awfully stiff.

Usability Critique

  1. [General] The website isn't responsive (or adaptive).
    • It's all laid out with HTML table elements, so nothing changes with the viewport.
  2. [Efficiency] There are too many categories, reducing click-through efficiency.
    • The sidebar categories are arranged alphabetically (i.e. without useful prioritization).
    • Hovering over the navbar reveals long lists of categories (figure 2a).
    • Clicking into some of these reveals even longer lists of subcategories (figure 2b).
    • Overall, this causes users to waste more time getting to the right category, instead of looking at products to buy. I suspect that this leads to lost revenue.

  1. [Learnability] Navigation is incongruous with the menu options' visual presentation.
    • Options (with the same text) on the sidebar and the navbar lead to the same pages. Yet, some options can only be found on the sidebar.
  2. [Memorability] Products featured on the home page change on refresh.
    • Also, there's no way to view all the featured products, so it's not easy to find "that one product you saw, when you last visited the page".

Accessibility Critique

WebAIM WAVE reveals over 270 errors! Most have to do with missing alt text, exacerbated by overuse of images where text, icons, and spacers should be. Other problems include:

  • Multiple duplicate links, which hinder quick screen reader navigation;
  • Low text contrast on the sidebar;
  • The inability to tab-navigate through navbar items; and
  • A missing lang attribute on the HTML element.

That said, the biggest issue by far is the website's overall HTML structure. It does not follow semantic HTML rules (e.g. using page regions like <nav>), nor does it contain any heading elements (e.g. <h1>).

Worse still, the whole page is laid out with tables instead of CSS, with an absurd number of hidden links, images used as text, and table cells that just have images of dividers.

Key Changes:
Note: points correspond to those from the usability discussion.

  1. The website is now responsive.
  2. Categories have been cut down. Each should lead to a page containing a list of products (which may then be filtered or sorted), not a list of even more subcategories.
  3. Contact numbers have been cut down to just one, at the bottom of the home page.
  4. Links have been re-styled to be consistent and salient.
    • Product panels are also now links, with hover interactions consistent with other links.
  5. Store navigation now uses categories and brands as top-level nav options.
    • Some users like to browse by category, others by brand. Having two "parallel" tracks helps funnel both types of users to where they can see (and buy) the products that they're more interested in.
    • Occasionally, users might instead bee-line it to a specific product. For these users, I made the search bar more conspicuous and efficient.
  6. Featured products are now static, and they make up much of the prime screen real-estate above the fold. There's also a "See All" link.
  7. Accessibility issues will also be addressed, though those can't be seen in a wireframe.
    • For instance, it'd be trivial to use nav, section, article, header, main, and footer elements in place of regular divs.

Conclusion

This project was, frankly, really time-consuming! The workflow I've described above, while useful for larger-scale redesigns, is probably overkill for as small an assignment as this one.

That said, I got to practice my usability analysis and iterative design skills, so it wasn't a total loss. I also have a far greater appreciation for the value of taking time to identify current flaws, so as to devise more targeted improvements. In all, this was a decently worthwhile exercise; and hopefully it'll help me design better websites in the future!
